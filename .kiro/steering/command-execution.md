# ターミナルコマンド実行ガイドライン

## バックグラウンドプロセス管理

長時間実行されるコマンド（開発サーバー、ビルドウォッチャーなど）には `controlBashProcess` を使用すること。短時間で完了するコマンドには `executeBash` を使用すること。

- `controlBashProcess` はバックグラウンドプロセスを適切に管理できる
- `executeBash` はシンプルだがバックグラウンドプロセス管理機能がない

## コマンド監視プロトコル

シェルコマンドを実行した後は必ず以下を行うこと：

1. `getProcessOutput` を使って実行状態を確認する
2. エラー、警告、完了シグナルの出力を監視する
3. エラーメッセージを分析してアプローチを調整し、失敗を適切に処理する
4. バックグラウンドプロセスの場合、次のステップに進む前に正常起動を確認する
5. **絶対に無限待機しない** — 以下のタイムアウト基準を厳守する：
   - 通常コマンド: 30秒
   - ビルド・インストール: 120秒
   - 開発サーバー起動確認: 15秒

## 「Working...」フリーズ防止ルール

以下の状況では **即座に次のステップへ進む**こと。待機を継続しない：

- `getProcessOutput` を呼び出して出力が空、または変化がない場合
- `listProcesses` でプロセスが `stopped` または存在しない場合
- 同じ `getProcessOutput` を2回呼び出して結果が同じ場合

**フリーズ検出時の対処手順：**
1. `listProcesses` でプロセス一覧を確認する
2. 対象プロセスが存在しない・停止済みなら、その結果をユーザーに報告して次へ進む
3. 「まだ実行中かもしれない」と推測して待機し続けない

## 環境エラー時の対応ルール

コマンド実行中に以下の環境系エラーが発生した場合は、**即座にユーザーに報告**してから一緒に解決すること。自分だけで解決しようとして時間を無駄にしない：

- Node.jsのバージョン不一致（`Unsupported engine`、`crypto.getRandomValues is not a function` など）
- パッケージマネージャーのバージョン管理ツール（asdf、nodebrew、nvm など）の設定問題
- `node_modules` が存在しない、または古いNodeバージョンでインストールされている
- シェルセッションが汚染されていてコマンドが正常に実行できない
- `npm install` がタイムアウトする

**報告フォーマット:**
> 🚨 環境エラーが発生しました
> **エラー内容:** （具体的なエラーメッセージ）
> **原因:** （推定される原因）
> **解決に必要な操作:** （ユーザーに実行してもらう手順）

## 重要な注意事項

- macOS (darwin / bash) 環境で動作している
- PowerShell系のコマンド (`controlPwshProcess`, `executePwsh`) は使用しない
- コマンドリスト (`&&`, `||`, `;`) は `executeBash` では使用しない — 複数コマンドは個別に呼び出す
- `cd` コマンドは使用しない — 代わりに `cwd` パラメータを使用する
